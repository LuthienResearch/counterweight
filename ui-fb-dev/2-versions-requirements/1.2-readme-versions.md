### Decisions

**Decided (facts, not requirements):**
- **GitHub README = landing page.** Jai decided Feb 5. No separate website. "Trying to maintain two landing pages is hard."
- **Demo video is the hero.** Jai + Chris + Beth Anne all want visual > text. Show Claude Code with Luthien intervening.
- **"Try in browser" is not the path.** Jai: "Too much work, may confuse." One-click cloud deploy (when SaaS is ready) is the pragmatic answer.

**Deferred (not README scope):**
- **One-click cloud deploy** (#18) + **pass-through auth** (#19)
- **Interactive playground** (#20)
- **Package safety** (#21)
- **Testing at scale** (#24)

## Synthesis: What the README should do

Based on 14 user feedback sessions. See [1-value-prop-feedback.md](../1-feedback-synthesis/1-value-prop-feedback.md) for full quotes and evidence.

### Bullseye target customer (Jai, Feb 7)

> "Forget about who pays you; find a human who loves what you've built." — Finn (Dec 2025)

**Must-haves:** Hands-on developer who uses Claude Code or Codex daily for production code. Has a detailed claude.md. Familiar with LLM APIs. English-speaking, ideally west coast US timezone. Has recently had an AI agent do something unexpected.

**Two personas, one bullseye:**

| Persona | Motivation | What they need from README |
|---------|-----------|---------------------------|
| **A: "Accelerate AI tooling"** | Maximize upside | Quick start (2 env vars), what policies exist, how to customize, "I can write my own" |
| **B: "Make sure AI tools don't cause problems"** | Minimize downside | What it catches, how blocking works, what happens next, "we handle the basics" |

### What "done" looks like

The README satisfies all 9 consolidated requirements in order: R1 (demo hero), R2 (plain English), R3 (two layers), R4 (block story), R5 (zero-friction setup), R6 (no AI slop), R7 (supported clients), R8 (security trust), R9 (funny/deadpan copy).

### What it should NOT do

- Don't show Python code above the fold
- Don't lead with a bulleted problem list
- Don't explain policy internals
- Don't show YAML config or SaaS management UI
- Don't list every feature
- Don't describe "the shape" before the value
