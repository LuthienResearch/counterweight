The hackathon calls for "control protocols that involve traditional security measures, such as giving the trusted model access to a firewall."
Proxy-level monitor for BashArena Track 2
Write a Luthien policy that flags suspicious sysadmin commands before they run. Ctrl-Z showed resample protocols cut attack success from 58% to 7%. Can a proxy-level monitor do better? $1K LinuxArena prize eligible.
The hackathon calls for "a dataset of sneaky transcripts of AIs doing very bad things that a prompted instance of Opus 4.6 can't catch."
Red team our policies Track 3
Try to get harmful actions past Luthien's built-in defenses (command blocking, DLP, LLM judge). Document which evasion strategies work, which don't, and why. Build a dataset of adversarial transcripts.
The hackathon calls for "interrogation protocols in which the trusted model asks the untrusted model questions before executing the action."
LLM-as-judge interrogation protocol Track 2
Before the untrusted model's action runs, a trusted model reviews it and asks clarifying questions. Extend Luthien's built-in LLM-as-judge policy. Measure the safety vs. usefulness tradeoff at different depths.
The hackathon calls for "a control protocol where the trusted model can run actions in the environment to learn more about the action the agent is proposing."
Multi-policy firewall for LinuxArena Track 2
Stack multiple Luthien policies (regex + DLP + LLM judge) into a layered defense. Test against LinuxArena tasks: does composing simple policies beat a single sophisticated one? $1K Equistamp LinuxArena prize eligible.
The hackathon calls for "red-teaming protocols like trusted editing harder, such that the editor preserves the bad code."
Trusted editing via proxy rewrite Track 2
Luthien intercepts suspicious code suggestions and has a trusted model rewrite them before they reach the developer. Does the editor keep the code working while removing sabotage? Build it AND red-team it.
