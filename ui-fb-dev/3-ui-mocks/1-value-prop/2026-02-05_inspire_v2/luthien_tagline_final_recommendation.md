# Luthien Tagline - Final Recommendation

## Top 3 Taglines (99% Confidence)

---

## #1: "Let AI code. Stay in control."

**The Winner.**

| Criterion | Score | Rationale |
|-----------|-------|-----------|
| Clarity | 5/5 | Immediately understandable |
| Emotion | 5/5 | Resolves anxiety about AI autonomy |
| Memorability | 5/5 | Parallel structure, rhythmic |
| Differentiation | 4/5 | "AI code" is specific; "control" is Luthien's space |
| Target Fit | 5/5 | Power users who trust AI but want oversight |

**Full hero block:**
```
Let AI code. Stay in control.

The drop-in proxy for AI coding agents. See every request,
enforce every rule, without changing your code.

[Get Started] [View on GitHub]
```

**Why it wins:**
- "Let" = permission, empowerment (not restriction)
- "Stay" = you already have control, Luthien maintains it
- Resolves the core tension: AI autonomy vs. human oversight
- Works for individual devs AND enterprise teams

---

## #2: "AI coding with guardrails."

**The Concise Alternative.**

| Criterion | Score | Rationale |
|-----------|-------|-----------|
| Clarity | 5/5 | "Guardrails" is a known metaphor |
| Emotion | 4/5 | Safety without restriction |
| Memorability | 4/5 | Short, concrete |
| Differentiation | 4/5 | "Guardrails" positions Luthien's niche |
| Target Fit | 4/5 | Appeals to safety-conscious devs |

**Full hero block:**
```
AI coding with guardrails.

Real-time visibility and rule enforcement for Claude Code,
Codex, and any AI coding agent. Open source.

[Get Started] [View on GitHub]
```

**Why it's strong:**
- Only 4 words—ultra-memorable
- "Guardrails" = safety without slowdown
- Implies you can still move fast
- Risk: "Guardrails" becoming overused in AI discourse

## #3: "Two env vars. Total visibility."

**The Technical Hook.**

| Criterion | Score | Rationale |
|-----------|-------|-----------|
| Clarity | 4/5 | Clear to devs, jargon to others |
| Emotion | 3/5 | More intellectual than emotional |
| Memorability | 5/5 | Extreme contrast is memorable |
| Differentiation | 5/5 | No competitor can claim this |
| Target Fit | 5/5 | Power users love concrete specifics |

**Full hero block:**
```
Two env vars. Total visibility.

Point your AI coding agent at Luthien. See every request,
every response, in real-time. Enforce rules. No code changes.

[Get Started] [View on GitHub]
```

**Why it's compelling:**
- Extremely concrete (builds technical trust)
- Tiny input → massive outcome
- Differentiates from competitors requiring SDKs/code changes
- Only works for technical audience (which is the target)

---

## Recommendation

**Primary:** "Let AI code. Stay in control."
- Use this on the landing page, GitHub README, and marketing

**Secondary:** "Two env vars. Total visibility."
- Use this in technical docs, developer-focused content, and CTAs

**Tertiary:** "AI coding with guardrails."
- Use this when "control" might sound restrictive (enterprise, compliance contexts)

## Stress Test: Objections

| Objection | Response |
|-----------|----------|
| "Control sounds limiting" | #1 uses "stay in control" (maintaining, not gaining) which is empowering |
| "Too similar to LangSmith" | All three emphasize AI *coding* specifically, not general LLM apps |
| "Guardrails is overused" | Valid concern; #1 avoids this |
| "Two env vars is too technical" | It's perfect for the target audience (senior devs); general audiences get #1 |
| "Doesn't mention open source" | Supporting text handles this; tagline stays focused |
