R8 Option A: Inline Config Fields

Policy customization via expandable config sections below each toggle

Design rationale: Extends the existing toggle UI (R5) with an expand/collapse config panel per policy. Users see toggles by default; click to customize. Minimal learning curve. Stays within the policy config page.

Tradeoff: Limited to pre-defined config fields. Can't create fully custom policies here; that still requires Python. Good for "80% customization" (which package manager, which commands to block).
