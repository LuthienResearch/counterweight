R8 Option B: YAML Config Editor

Power-user policy customization via direct YAML editing (like VS Code settings.json)

Design rationale: Mirrors the developer mental model: policies are YAML config. Shows the actual config file with syntax highlighting. Users who already edit policy_config.yaml will feel at home. Supports full flexibility including creating custom policies.

Tradeoff: Higher learning curve. Non-technical users won't use this. But our users ARE senior developers (Tyler, Quentin, Nico) who edit YAML daily. Diff view shows what changed vs defaults.

Key question: Is this too close to "just edit the file"? The value-add is: (1) validation before save, (2) diff vs defaults, (3) no restart needed.
