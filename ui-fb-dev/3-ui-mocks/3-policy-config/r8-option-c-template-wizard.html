R8 Option C: Template-Based Policy Creator

Guided flow: pick a template, fill in your parameters, deploy

Design rationale: Users pick from pre-built templates, then customize parameters via a guided form. Covers both modification (edit an existing policy's settings) and creation (new policy from template). Progressive disclosure: templates show "what kinds of policies exist" (R4/Finn's question) while the form lets users customize (R8/Quentin's need).

Tradeoff: More complex UI than Options A or B. But it's the only option that explicitly addresses policy CREATION (not just modification). Good for the "expand what's possible" use case. Templates can be community-contributed over time.
