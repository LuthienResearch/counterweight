v8: Inspiration-Driven Redesign

v7 was organized around Luthien's architecture (policies, toggles, config). v8 redesigns from the user's mental model: "is my code cleaner/safer?" The before/after demo does 90% of the work. Everything else is infrastructure users tolerate, not value they seek.

Analogies

Each concept steals from products that "just work" — where value is obvious and config is optional.

Thermostat
Steal: One-number status. Set and forget.
Delete: Rule builders, condition/action forms
Ad Blocker
Steal: Works on install. Counter = proof.
Delete: Tooltips, testing sandbox, bulk actions
Car Warning Lights
Steal: Silent until something happens.
Delete: Always-visible dashboards
Gmail Filters
Steal: Create rules from observed events.
Delete: Import/export, YAML editors

DeSlop is always the star. Every mockup leads with it as the default active policy.

Concept 1 The Scoreboard

Your code quality has a score, and it keeps going up.

One big number dominates the screen. A reverse-chronological feed shows every catch. The most recent catch shows the full before/after diff inline. Older catches collapse to one-liners. Policy management lives in a drawer at the bottom — accessible but never front-loaded.

Scoreboard Dashboard

Terminal
Running Counter
Inline catch count, session exit scoreboard
Concept 2 The Diff Viewer

Every request is a git diff. Luthien shows what it changed and why.

Split-pane view: raw LLM output on the left, after Luthien on the right. Annotations on changed lines show which policy fired. Click an annotation to edit that rule — the "Gmail filter from example" pattern. You see what happened, then configure from context.

Browser
Split-Pane Diff
Side-by-side diff with clickable policy annotations
Terminal
git diff --stat Style
Inline summary, /luthien diff for full view
Concept 3 Terminal Native

Luthien is a shell command, not an app. Config lives in CLAUDE.md.

The terminal is the full experience. The browser is just a read-only activity mirror for demos and people who prefer pictures. Install, configure, forget. /luthien gives you an interactive menu mid-session. Config-as-code in YAML.

Browser
Live Activity Mirror
Read-only view of what's happening in the terminal
Terminal
The Full Experience
